# Project Title: Counter Baseball

## 📖 Description
투수가 반짝이는 모션을 보고 공이 도달하기 전에 'space'를 눌러서 성공적으로 카운터를 쳐야 하는 반응형 야구 게임입니다.
레벨이 올라갈수록 투구 속도가 빨라지며, 그만큼 빠르게 반응해야만 성공할 수 있습니다.

## 🧠 논리적 설계
### 필요 리소스 이미지
- 투수 (스탠딩, 다리 한 쪽 드는 준비자세, 공을 던진 폼)
- 타자 (스탠딩, 휘두르기 직전 자세, 공을 때리는 폼)
- 야구장 배경
- START 버튼
- 노란 동그라미 (스트라이크)
### 필요 리소스 사운드
- 야구장 BGM (응원소리)
- 투수 던지는 소리 (로스트아크, 보스 카운터 bgm)
- 타자 타격 소리 (로스트아크, 카운터 성공 bgm)
### 게임 동작 프로세스
1. 게임 시작 버튼을 누른다
2. 점수 0점, 공 속도 1, 스트라이크 카운트 0로 설정
3. 투수는 공을 3-10초 사이에 랜덤하게 던지기 시작한다
4. 투수의 폼이 준비자세로 바뀌며 소리가 던지는 이벤트가 발생한다 (공의 속도가 높아질수록 해당 폼의 지속시간이 줄어든다)
5. 투수의 폼이 던진자세로 바뀌며 공이 날아간다
    6. if 준비자세 시점부터 공이 지나가기 전까지 space를 누르면 [홈런] 으로 간다
    7. else if 준비자세 전에 space를 누르면 [헛스윙] 으로 간다
    8. else if 공이 지나가면 [헛스윙] 으로 간다
- [홈런]
1. space가 2초 동안 동작하지 않는다 (연속 스윙 방지)
2. 투수가 공이 도달하는 타이밍에 맞게 공을 쳐서 공이 날라간다 (타격 소리 on)
3. 점수 +1, 공의 속도 +1
4. 메인의 3번줄로 돌아간다
- [헛스윙]
1. space가 2초 동안 동작하지 않는다 (연속 스윙 방지)
2. 투수가 빠르게 스윙한다 (공은 맞지 않는다)
3. 스트라이크 카운트 +1
4. if 스트라이크 카운트가 3이면 게임이 종료된다
5. 메인의 3번줄로 돌아간다

## 🔨 개발 피드백

### 첫 대실패, 역할 분담을 '명확히' 해야 됨
"대략적인 게임 진행 순서를 머리 속에 인지하고 있는 상태"로 개발을 시작했다.
그러나 처참하게 깨지고 만다. 스프라이트 별로 코드들이 따로 놀며, 내가 원하던 기능을 구현하기란 생각보다 쉽지 않았다.
또한 "스크래쳐"라는 제한 속에서 어떤 도구가 있었는지도 모르기 때문에.. 생각하던 기능을 구현하려면 코드가 너무 지져분해졌다.
결국 쓰레기 값들이 많아지고 코드는 난잡해지고.. 나는 "완전 리셋"이라는 결단을 내리게 됐다.
꼭 소득이 없는 것만은 아니였다. 스프라이트들의 움직임을 어떻게 구성해야 되는지, 특히 "Control" 블록들의 가능성과 제약이 무엇인지,
이러한 것들을 파악할 수 있는 시간이었다. 구현해놓은 모든 블록을 초기화했다.
또한 강의에서 Ai를 사용하지 말라고 했다. 그 말 자체가 "Ai가 과제를 해주면 안 된다"라는 뉘앙스임은 알고 있었지만, 블록의 기능을 묻는 것조차 하지 않고 있었다. 이것은 결코 개발자나 해커가 가질만한 마인드셋이 아니다. 검색엔진의 활용을 (과제를 해주는 것은 절대 아닌 선에서) 더욱 적극적으로 해보도록 하자.

### 실패 이후, 코드를 시작하기 전 흐름 설계부터
나는 기존에 C 언어로 아주 기초적인 코딩만 몇 번 해봤다. C에선 훨씬 스크래쳐보다 자유도가 높았던 것을 기억한다.
그래서 C가 가능한게 다 가능할 것이라 생각했지만, 그 기능을 구현함에 있어서는 상당히 불편함을 느꼈다. 
짧게나마 스크래쳐를 다뤄본 기억을 바탕으로 기본적인 흐름을 다시 세팅하고 가고자 함.

1. Green Plag 동작 시, broadcast(Reset) 을 통해 각 스프라이트들을 '초기 상태'로 만든다
2. PLAY 버튼을 클릭 시, broadcast(Start)로 스프라이트들이 등장하며 바로 1라운드가 시작된다
3. 휘슬이 울리면 Ingame 카운트가 1이 되며, space가 동작이 가능해진다 (Ingame이 아닐 땐 동작하지 않음)
4. 투수가 공을 랜덤한 타이밍에 던진다(홈런 타이밍은 Score에 비례해 줄어든다)
    5. 공을 던지기 전에 휘두르면 바로 휘두르고 공은 제 때 날아간다, Strike++ 
    6. 정확한 타이밍에 휘두르면 공에 맞고 홈런이 된다, Score++
    7. 늦은 타이밍에 휘두르면 공이 먼저 들어가고 휘두른다, Strike++
    8. 휘두르지 않고 Ingame이 0이 되면, Strike++
9. if(Strike>2) 게임이 종료되며 1번으로 돌아간다 (그린플래그 동작과 동일한 Reset 상태)
10. if(Score==10) 승리하며 게임이 종료되며 1번으로 돌아간다
10. 4번으로 돌아간다
